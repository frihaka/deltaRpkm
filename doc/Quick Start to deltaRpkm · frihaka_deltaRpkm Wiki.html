Quick Start to deltaRpkm

Jump to bottom Edit New Page
frihaka edited this page Feb 27, 2019 · 12 revisions

1 - deltaRpkm

This is an R package whose main purpose is to quickly identify genes potentially involved in a given phenotype by performing a differential analysis of genes coverage between two sets of closely related genomes (reference genome group 1 and comparison group 2).
The package provides functions to compute:

  1. RPKM,
  2. Œ¥RPKM,
  3. selection of genes that are differentially present in the reference genome group 1
  4. heatmap plot to highlight the clustering of samples groups being compared, based on the RPKM values of the filtered genes

It also includes methods to perform some batch effects controls and diagnostics plots.

2 - Pre-requisites

  1. R/Rstudio, minimum familiarity with R ecosystem (objects, working environment, installing packages from CRAN and Bioconductor)
  2. R dependencies: from CRAN (ggplot2, ggfortify, dplyr) and Bioconductor (sva, Biostrings)
  3. Two input files: reads counts table and metadata table (see working examples below to check out for the formats)

Install deltaRpkm

Download the binary from GitHub.
Install either from terminal via:

R CMD INSTALL deltaRpkm_0.1.0_R_x86_64-pc-linux-gnu.tar.gz  # assuming no missing R dependencies, e.g ggplot2 etc

or from within R via:

> install.packages("deltaRpkm_0.1.0_R_x86_64-pc-linux-gnu.tar.gz",  
                   repos = NULL,
                   dependencies = TRUE)  # this makes sure that CRAN packages are installed if missing from your system

In any ways, Bioconductor packages sva and Biostrings will need to be installed if not already in the system.

3 - Load datasets

The package provides some examples datasets to get the hands on.

Load a coverage table

library("deltaRpkm")
library("ggfortify")

> data("coverage_table_N51")

Load a metadata table

> data("metadata_table_N51")

Format the metadata table with loadMetadata():

> design_table <- loadMetadata(user_metadata = metadata_table_N51,   
                             delta_phenotype_colname = "lineage_type", 
                             heatmapbar_phenotype_colname = "infection", 
                             samples_colname = "sample", 
                             genome_length_colname = "genome_length",
                             mapped_reads_colname = "mapped_reads")

4 - Convert read counts to RPKM

Format the metadata table with rpkm():

> rpkmtable <- rpkm(user_metadata = design_table,
                  coverage_table = coverage_table_N51,               
                  delta_phenotype_colname = "lineage_type",      
                  heatmapbar_phenotype_colname = "infection")

5 - Compute deltaRPKM values

The difference in RPKM values for a given gene, between samples of group 1 and group 2 are calculated with deltarpkm():

> deltarpkm_table <- deltarpkm(rpkm_table = rpkmtable,                          
                             genes_names = unique(rpkmtable$geneID),              
                             samples_colname = "sample",
                             delta_phenotype_colname = "lineage_type",          
                             reference_sample = "JF5203",             
                             nonref_delta_phenotype = "Lineage_II")

6 - Identify differentially present genes

Strategy

The strategy is to compute the:

  1. m : median deltaRPKM value per gene
  2. s : standard deviation of all the median deltaRPKM values
  3. differential gene presence significance threshold, arbitrarily chosen by default as 2*s
  4. selection of the genes that present a median deltaRPKM value m>=2*s

This can be done with the function deltaRPKMStats():

> stats_table <- deltaRPKMStats(deltarpkm_table = deltarpkm_table)

Visual check of the median deltaRPKM (m) values distribution

> median_plot(data_table = stats_table, 
	    coverage_table = coverage_table_N51[, c("start", "end", "geneID")])

This creates a deltaRpkm_medians_plot.pdf file in the working directory:

Select the genes that are differentially present in the reference genome group 1

> differential_present_genes <- unique(stats_table[stats_table$selected_gene %in% "+", ]$geneID)

7 - Heatmap of RPKM values clustering from the selected genes

Select relevant RPKM values

The RPKM table can be subsetted with subsetRPKMTable():

> heatmap_table <- subsetRPKMTable(rpkm_table = rpkmtable,
                                 user_metadata = design_table,
                                 delta_phenotype_colname = "lineage_type",
                                 heatmapbar_phenotype_colname = "infection",
                                 sd_filtered_genes = differential_present_genes)

Convert the subsetted RPKM tables to a matrix

Use convertHeatmapToMatrix():

> heatmap_matrix <- convertHeatmapToMatrix(wide_rpkm_table = heatmap_table,
                                         delta_phenotype_colname = "lineage_type",
                                         heatmapbar_phenotype_colname = "infection")

Plot the heatmap

With the function rpkmHeatmap():

> rpkmHeatmap(filtered_rpkm_matrix = heatmap_matrix,                  
            user_metadata = design_table,
            heatmapbar_phenotype_colname = "infection")

This creates an output figure deltaRpkm_heatmap.tiff in the working directory:

8 - Miscellaneous

Fine tune heatmap color breaks

One can also play with the color breaks of the heatmap by adjusting the parameters in the rpkmHeatmap() function. The idea here is to adjust the colors to the samples/genes clustering in the heatmap, based on the RPKM values bimodal distribution by selecting the boundary RPKM values around the main peak:

> hist(rpkmtable$rpkm)

The default settings and the peak boundaries used in rpkmHeatmap() function can be checked by interrogating the package documentation:

> ?rpkmHeatmap

Note how the default peak boundaries are given as lower_limit = 300 and upper_limit = 550.

One can adjust these values empirically (trial/errors) based on the RPKM histogram distribution above. Or use one of the methods provided by deltaRpkm as boundaries():

> res <- boundaries(x = rpkmtable$rpkm, strategy = "mclust")
> res$boundaries_df
   boundaries rpkm_values
1 lower_limit         300
2 upper_limit         624
> res <- boundaries(x = rpkmtable$rpkm, strategy = "ratios") 
>    boundaries rpkm_values
1 lower_limit         295
2 upper_limit         585

> res <- boundaries(x = rpkmtable$rpkm, strategy = "quartiles") 
>    boundaries rpkm_values
   boundaries rpkm_values
1 lower_limit         383
2 upper_limit         487
